Discuss the scope of var, let, and const

var declarations are globally scoped or function scoped while let and const are block scoped. var variables can be updated and re-declared within its scope; let variables can be updated but not re-declared; const variables can neither be updated nor re-declared

Tell us the use cases of null and undefined

1.Null
  1. Explicit Absence of Value: It is used to explicitly indicate the absence of a value or that a variable intentionally has no value.
  2. Clearing Object Properties: Developers can set an object property to null to indicate that it should be considered empty or no longer in use.
  3. Comparison: null is often used when comparing or checking for the absence of an object or value.
  4. Garbage Collection: In some memory management scenarios, setting a reference to null can help in releasing memory resources by indicating that an object is no longer needed
1.Undefined
  1. Default Initial Value: When a variable is declared but not initialized, it is assigned the value undefined by default.
  2. Function Parameters: If a function is called with fewer arguments than declared parameters, the missing parameters are set to undefined.
  3. Object Properties: If you try to access an object property that doesn't exist, the result is undefined.
  4. Return Value: Functions that don't explicitly return a value return undefined

What do you mean by REST API?

REST API stands for "Representational State Transfer Application Programming Interface." It is an architectural style for designing networked applications, particularly web services, that relies on a stateless communication protocol, typically HTTP. REST is not a technology or a standard but rather a set of constraints and principles for building scalable and maintainable web services.
